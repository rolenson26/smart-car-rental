# Database Design Document

## User

- `id`: Primary key.
- `name`: User display name.
- `email`: Unique login email.
- `password_hash`: Secure password hash generated by Werkzeug.
- `role`: `user` or `admin`.
- `phone`: Optional contact number.
- `created_at`: Registration timestamp.

Relationships:

- One user has many bookings.
- One user has many reviews.

## Vehicle

- `id`: Primary key.
- `vehicle_name`: Public listing name.
- `category`: Car, Bike, Scooter, Van, or another category.
- `brand`: Manufacturer.
- `model`: Model name.
- `registration_number`: Unique vehicle registration identifier.
- `rental_price`: Daily rental price.
- `image`: Listing image URL.
- `availability_status`: Available, Maintenance, or Unavailable.
- `seats`: Passenger capacity.
- `fuel_type`: Petrol, Diesel, Electric, Hybrid, etc.
- `transmission`: Manual or Automatic.
- `description`: Listing description.
- `created_at`: Listing creation timestamp.

Relationships:

- One vehicle has many bookings.
- One vehicle has many reviews.

## Booking

- `id`: Primary key.
- `user_id`: Foreign key to `users.id`.
- `vehicle_id`: Foreign key to `vehicles.id`.
- `booking_date`: Rental start date.
- `return_date`: Rental return date.
- `status`: Pending, Confirmed, Active, Completed, or Cancelled.
- `total_amount`: Calculated rental amount.
- `created_at`: Booking creation timestamp.

Relationships:

- One booking belongs to one user.
- One booking belongs to one vehicle.
- One booking has one payment.

## Payment

- `id`: Primary key.
- `booking_id`: Foreign key to `bookings.id`.
- `amount`: Amount due for the booking.
- `payment_status`: Pending, Paid, Failed, or Refunded.
- `created_at`: Payment record timestamp.

## Review

- `id`: Primary key.
- `user_id`: Foreign key to `users.id`.
- `vehicle_id`: Foreign key to `vehicles.id`.
- `rating`: Integer rating from 1 to 5.
- `comment`: Optional review comment.
- `created_at`: Review timestamp.
